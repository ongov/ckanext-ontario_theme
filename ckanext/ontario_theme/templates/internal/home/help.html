    Colby is in alpha release, which means it is a prototype that we are using to run experiments on how to increase data sharing and collaboration. There is no draft mode yet for data—everything is be visible to the entire OPS immediately upon saving. Don’t share data files until they have been approved, but you can still add metadata while you’re waiting.

    How to archive data

    There is no need to archive data. Instead, continually add new data when it is collected and making it clear to others what time series they are viewing.

    When you have data that is in a time series (for example, collected annually or reflecting an annual cycle of work), ensure that your file names include the time series if the data will be collected in separate files. You can also collect all time series in the same file and update the file, as needed.

    Open Data Guidebook and Directive

    The guidebook provides direction on how to identify data, assess risks, prepare Open Data for approval and publish datasets. It includes several resources, ideas for open data engagement, links to useful templates and answers to frequently asked questions.

    Ontario's Open Data Directive maximizes access to government data by requiring all data to be made public, unless it is exempt for legal, privacy, security, confidentiality or commercially-sensitive reasons. It sets out key principles and requirements for publishing open data, and applies to data created and managed by Ontario ministries and provincial agencies.

    Government-Wide Data Inventory

    The government-wide data inventory was established in 2016 and keeping it current is an ongoing responsibility of Ontario's ministries and provincial agencies.

    A data inventory is a list of datasets under the custody and control of the organization. Each ministry and provincial agency must create and maintain an inventory of their datasets according to standards set out by TBS.

    How to format basic numeric data in spreadsheets

    Data is most useful when it can be read by computers. It’s easier to re-use, mash up with other data and analyze. It can also be quickly turned into charts or graphs. Follow these guidelines to maximize the usefulness of your data!

    1. Don’t merge cells. Sorting and other manipulations people may want to apply to your data assume that each cell belongs to one row and column. Merging cells also makes the sheet inaccessible to people using screen readers.
    2. Don’t mix data and metadata (e.g. date of release, name of author) in the same sheet. Colby has a form specifically for entering metadata when you add new data.
    3. Put only column headers in the first row of a data sheet. Don’t include duplicate or blank headers. Do include units of measure where applicable (for example, kilometers, millions of dollars, etc.).
    4. The remaining rows must be data with only one set of information per row. Don’t include totals or averages in the main sheet. Do include totals and averages in a separate sheet, if they are important. Alternately, use a smartie or a graph to display totals and averages in the data description on Colby.
    5. Format numeric values as plain numbers. Don’t put commas in them, dollar signs, asterisks or anything else. If you need to add an annotation, do it in a separate column.
    6. Use standard identifiers (for example, identify countries using ISO 3166 codes rather than names, use ISO 8601 / GO-ITS 74.00 format for dates, etc.).
    7. If you also want to create a human-friendly version of the data, create another sheet in the same workbook or contact Open Government to discuss how best to tell your data story!

    How to convert spreadsheets to CSV

    In Colby, you can display data in a graph or a map if you make it into a machine-readable CSV file in UTF-8 format. Here’s how to do that:

    1. Click File > Save As and change the encoding to UTF-8. Then click the Save button.
    2. Use an online file conversion tool (for example docspal or convertio) to change the XSLX to CSV. Don’t use online converters for data that is not open (they aren’t secure).

    How to get updates from Colby using RSS

    There are five main feeds in Colby that show recently updated or created data. Add one of these URLs to a RSS feed reader and you'll get updates when they happen.

    1. All data updates on Colby: https://intra.catalogue.ogo.tbs.gov.on.ca/feeds/dataset.atom
    2. Updates to specific data (change the text after id= in the URL to match the data you want to monitor): https://intra.catalogue.ogo.tbs.gov.on.ca/feeds/dataset.atom?id=zooplankton-monitoring
    3. Updates and new data for a specific ministry (change the name in the URL to match the ministry you want to monitor): https://intra.catalogue.ogo.tbs.gov.on.ca/feeds/organization/agriculture-food-and-rural-affairs.atom
    4. Updates and new data for a specific tag (change the tag in the URL to match the tag you want to monitor): https://intra.catalogue.ogo.tbs.gov.on.ca/feeds/tag/tag-here.atom
    5. Updates and new data for a specific search query (change the text after q= in the URL to match the search query you want to monitor): https://intra.catalogue.ogo.tbs.gov.on.ca/feeds/custom.atom?q=apple
